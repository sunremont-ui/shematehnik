# Code Generator Module

## Purpose

Генерация кода для встроенных систем: CRC-калькулятор, LVGL UI-экспорт, Arduino/ESP-IDF скетч-генератор и генератор кода протокольных пакетов.

## Module Tree

```
CodeGenModule
├── CrcCalculatorModule     ← вычисление CRC + генерация C-кода таблицы
├── LvglExportModule        ← генерация LVGL C-кода экранов
├── ArduinoExportModule     ← Arduino .ino / ESP-IDF main.c
└── ProtocolCodeGenModule   ← encode/decode с CRC для пользовательских пакетов
```

Зарегистрированные типы: `CodeGenModule`, `CrcCalculatorModule`, `LvglExportModule`, `ArduinoExportModule`, `ProtocolCodeGenModule`

## CrcCalculatorModule

### Поддерживаемые алгоритмы CRC

| Алгоритм | Bits | Poly | Init | RefIn | XorOut |
|----------|------|------|------|-------|--------|
| CRC-8 | 8 | 0x07 | 0x00 | false | 0x00 |
| CRC-8/MAXIM | 8 | 0x31 | 0x00 | true | 0x00 |
| CRC-16/IBM | 16 | 0x8005 | 0x00 | true | 0x00 |
| CRC-16/CCITT | 16 | 0x1021 | 0xFFFF | false | 0x00 |
| CRC-32 | 32 | 0x04C11DB7 | 0xFFFFFFFF | true | 0xFFFFFFFF |
| CRC-32/BZIP2 | 32 | 0x04C11DB7 | 0xFFFFFFFF | false | 0xFFFFFFFF |

### Форматы ввода

- **HEX** — `AA BB CC DD` (байты через пробел)
- **ASCII** — текстовая строка
- **Decimal bytes** — `170 187 204 221`

### Генерация C-кода

Кнопка **Generate Code** выводит полный C-файл с lookup-таблицей:

```c
/* CRC-16/CCITT lookup table */
static const uint16_t crc16_table[256] = { ... };

uint16_t crc16_ccitt(const uint8_t *data, size_t len) {
    uint16_t crc = 0xFFFF;
    for (size_t i = 0; i < len; i++)
        crc = (crc << 8) ^ crc16_table[(crc >> 8) ^ data[i]];
    return crc;
}
```

## LvglExportModule

### Структура экрана

```
ScreenConfig
├── name        → "main_menu"
├── title       → "Main Menu"
└── widgets[]
    ├── {type="label", id="lbl_title", text="Hello"}
    ├── {type="btn",   id="btn_start", geom=QRect(10,50,100,40)}
    └── {type="chart", id="chart1",    geom=QRect(0,100,480,200)}
```

Поддерживаемые типы виджетов: `label`, `btn`, `list`, `chart`, `bar`.

### Генерируемые файлы

| Файл | Содержимое |
|------|-----------|
| `screens/main_menu.c` | `lv_obj_t*` объявления + lv_create_* вызовы |
| `main.c` | `lv_init()` + вызовы инициализации экранов |
| `CMakeLists.txt` | Минимальный ESP-IDF / PlatformIO билд |

### Настройки дисплея

Разрешение задаётся spinbox-полями Width / Height (по умолчанию 480×320).

## ArduinoExportModule

Генерирует скетч из выбранной программы (ProgramSystem).

### Целевые платформы

| Цель | Формат | Файл |
|------|--------|------|
| Arduino | `.ino` | `sketch.ino` |
| ESP-IDF | `main.c` | `main/main.c` |
| PlatformIO | `main.cpp` | `src/main.cpp` |

### Структура скетча (Arduino)

```cpp
// Generated by UCP — [program name]
#define PIN_PUMP   D2
#define PIN_SENSOR A0

void setup() {
    pinMode(PIN_PUMP, OUTPUT);
    Serial.begin(115200);
}

void loop() {
    // program logic
}
```

Pin mapping берётся из ProgramSystemModule → PinMappingWidget.

## ProtocolCodeGenModule

Генерирует C encode/decode для пользовательских пакетов (аналог Kaitai Struct для embedded).

### Определение пакета

1. Введите имя пакета (например, `SensorData`)
2. Добавьте поля через кнопку **+ Field**:
   - имя, тип (`uint8_t`, `int16_t`, `float`, `uint32_t`), описание
3. Выберите CRC-алгоритм (опционально)
4. Нажмите **Generate**

### Генерируемый код

```c
/* Auto-generated by UCP Protocol Code Gen */
#pragma pack(push, 1)
typedef struct {
    uint8_t  device_id;
    int16_t  temperature;   /* × 10, e.g. 215 = 21.5°C */
    float    humidity;
    uint16_t crc;
} SensorData_t;
#pragma pack(pop)

int SensorData_encode(const SensorData_t *in, uint8_t *buf, size_t buflen);
int SensorData_decode(const uint8_t *buf, size_t len, SensorData_t *out);
```

Функции encode/decode включают CRC-проверку если алгоритм выбран.

## Keyboard Shortcuts

| Контекст | Клавиша | Действие |
|----------|---------|---------|
| CRC Calc | `Ctrl+Enter` | Рассчитать |
| Все | `Ctrl+S` | Экспортировать в файл |
| ProtocolCodeGen | `+` / `Del` | Добавить / удалить поле |

- Qt6::Widgets
- Нет внешних зависимостей (CRC вычисляется встроенным `CrcEngine`)
