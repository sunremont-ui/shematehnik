# Philosophy

## Принципы

### 1. Модульность как дерево

Не плоский список плагинов, а **дерево**: модуль может содержать подмодули, те — свои подмодули. Это отражает реальную структуру проекта:

```
Проект
  └── Печатная плата
        ├── Схема
        │     ├── Компоненты
        │     └── Соединения
        └── 3D-корпус
              ├── Детали
              └── Сборка
```

Каждый узел дерева — самостоятельный модуль со своим UI, сохранением и логикой.

### 2. Всё — open source

- Ядро, модули, библиотеки — только открытый код (C++17, Qt6, OpenCASCADE, ngspice)
- Никаких платных лицензий
- Формат проекта `.ucp` — открытый JSON

### 3. Event-driven архитектура

Модули не вызывают друг друга напрямую. Они общаются через **EventBus**:

```cpp
// Модуль A шлёт событие
EventBus::emit("netlist.updated", data);

// Модуль B подписан
EventBus::on("netlist.updated", callback);
```

Это позволяет добавлять/удалять модули без изменения кода соседей.

### 4. Source of truth — схема

В центре проекта — **принципиальная схема**. Из неё генерируется:
- Netlist для SPICE-симуляции
- PCB (трассировка)
- 3D-модель (размеры компонентов)
- Документация

### 5. Человек управляет, машина делает

- Программист/инженер задаёт **что** сделать
- Программа делает **как** (трассировка, генерация кода, симуляция)
- Человек проверяет результат и правит

### 6. Zero lock-in

- Экспорт в стандартные форматы: Gerber, STEP, PDF, C/MicroPython
- Никакого проприетарного бинарного формата
- Модули — это C++ классы, не обфусцированные плагины
